This workshop is a basic introduction to front-end coding, which creates the visible visual elements of a website such as buttons, text, colours, and image placement.

Front-end code vs back-end code

Front-end development is also called client-side development. Basically this means that it makes what the user sees and interacts with.

Website development is split into two kinds of code - we call it front-end development and back-end development. Front-end development is made up of coding languages such as HTML, CSS, and Javascript, and back-end development has a multitude of languages. I’m a front-end developer so that’s going to be the focus of the workshop. It takes both to make a functioning, interactive website.

HTML Hyper Text Markup Language

CSS Cascading Style Sheets

Javascript Commonly shortened to JS. Scripting language, mainly used in websites but also used in web apps and some phone apps and games. Can manipulate HTML and CSS.

CMS Content Management System

Building blocks of code

Building your HTML and making it look nice with CSS is a little bit like building a house.

The designer and architect — web designer and user experience (UX) designer — draw out and design the look and feel of the house (web design), as well as creating blueprints (wireframes).

User Experience Designer creates outline of website, Web Designer uses that outline and makes it look nice.

Then, we build the frame of the house, which is HTML. HTML lays out the foundation and structure of the website.

HTML lays out the structure of the website.

CSS tells the HTML what to look like. It can change an element's colour, size, position, shape, and more. CSS is the decorating of the house - everything from wall colour to roof tile to shape of the windows. CSS can also handle minor interactions like changing a button's colour when you hover over it, but for more user advanced interaction, we would need to use Javascript.

CSS tells HTML what to do. Usually it's to make it look good. (CSS got HTML's back.)

To complete the analogy, the back-end code would be the electricians, plumbers, etc. coming in to put in wiring for electricity to the light switches and pipes so that you would have running water when you turn on the faucet.

There are also other components to creating your own website and putting it online, just like how there are prerequisites to owning a house. You would have to purchase a plot of land to place your house (a host to host your website) and an address so that people can get to your cool digs (a domain, like a .com or .ca).

The HTML structure is the same but CSS can tell the elements to move wherever.

Let's start

We're going to be using a website called codepen to learn the basics of setting up an html site and making it look nice with some css. An account isn't necessary but feel free to create one - it allows you to save your projects, fork off of others and to save your favourite projects from other people.

Before we start, let's go over how a real HTML page is structured. Because codepen does all of this for you, we're not going to worry about the setup, but understanding how the page is put together is going to make more sense for when we style things with CSS later on.

Tags are used in HTML and are always structured in pairs - opening and closing tags. The only difference between the opening and closing tag is a slash before the tag name.

HTML files are created as page.html

In codepen, all of the setting up is done for you so basically you are putting your content where it's supposed to be - between the <body> tags.

CSS is structured differently than HTML as it's a different language, but we're going to talk about that in a bit. For now, we need to know that for the browser to understand that we want the CSS to interact with your HTML, we have to add the file into the HTML page structure. This goes between the <head> tags and everything between there is hidden from the user. Codepen does all of this for you automatically as well!

CSS files end with .css - usually we name them style.css or main.css

Inspiration + resources

Free Learning

  • codeacademy

    Codeacademy is probably one of the best free resources for learning both front end code and back end code.

  • khan academy

    Khan Academy has some good introductory courses for HTML/CSS, as well as some more advanced scripting and database stuff.

Paid Courses and Classes

  • lighthouse labs

    Coding boot camp, with instructors and locations across Canada and helps with job placement upon course completion.

  • treehouse

    I've used Treehouse briefly, you learn from a series of videos and you can download the project to follow along with the video tutorial. Not really my learning style but works for some people.

Tools + Programs

  • brackets

    Brackets is a code editing program that is easy to setup and run, and you can see your code right away. Also has a preview mode so you can see what you’re doing in the browser.

  • atom

    Atom is another code editing program - there's a little more setup involved with it but it has many add-ons made by the community that you can customize with (for example, showing hex colours or cleaning up code). It's my current code editor of choice.

  • google fonts

    Google Fonts is a great resource for putting custom fonts in your code without having to download them or host them yourself. And they're free!

  • codepen

    Codepen is both a fantastic tool for experimenting with code (we did our workshop in codepen) and a great place for inspiration, as you can see how others are putting together their code. Needs no setup and shows your changes immediately - it's not a complete replacement for a code editor if you are going to be making your own website, but it makes a great starting point.

  • canadianwebhosting

    There are many hosting sites out there, but I use Canadian Web Hosting for my portfolio site. It's good for small sites and cheap.

  • namecheap

    I also use namecheap to buy domains (sometimes they have really good sales) and they also offer hosting, so you can get it all in one place, which makes it a little easier.

Inspiration / Cool Things

  • smashing magazine

    Smashing Magazine is a fantastic resource for all things web, ranging from news and updates on web libraries to design inspiration and cool code tricks to free icons and templates.

  • codrops

    Codrops is one of my favourite go-to websites for code and interactive inspiration, as well as tutorials.

  • css tricks

    CSS Tricks has news and updates on the newest code tools, as well as snippets and tutorials.
